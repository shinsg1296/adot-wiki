# 의사결정: 조직 구조 (운영 6부서 + 본사 레이어 + 분담 규칙)

## Summary

에이닷의 AI 에이전트 조직을 **운영 레이어(6부서) + 0_본사(전략실) 레이어**의 2층 구조로 확정. 부서 간 책임 경계를 규칙 R1~R3으로 고정.

## Context

에이닷의 목적 = 미래 사업 런칭 전 리스크를 최소화해 **운영 시스템과 원장 역량을 검증하는 연습장**. (에이닷 본사가 직원을 키워 원장 역량자를 앉히는 모델을 원장 자신에게 적용.) 따라서 진짜 산출물은 '자동화된 학원'이 아니라 ① 이식 가능한 운영 플레이북 + ② 성장한 운영자. → 구조도 '이식층 vs 도메인층'으로 분리되어야 함.

## Details

### 운영 레이어 6부서
[[1_학습관리부]] 수업일지·성적 / [[2_교무부]] 진도·자료·숙제·DB콘텐츠 / [[3_상담소통부]] 통화·상담·알림 / [[4_행정운영부]] 원생·출결·출력 / [[5_시스템개발부]] 도구·자동화·DB기술 / [[6_브랜드홍보부]] HAWE·홍보·신규문의(신설)

### 0_본사(전략실) 레이어 — 신설
운영 위에서 시스템 자체를 키움: 플레이북화·원장 육성·디리스킹/런칭 준비. 운영↔본사 양방향 루프. → [[0_본사_전략실]]

### 분담 규칙
- **R1 도구 소유**: 엔진(도구 빌드·유지) = [[5_시스템개발부]] / 산출물 '내용' = [[2_교무부]]·[[1_학습관리부]]
- **R2 DB 책임**: 콘텐츠(무엇을 넣나) = [[2_교무부]] / 기술(병합·스키마·푸시) = [[5_시스템개발부]]
- **R3 외부 채널**: 외부·마케팅 = [[6_브랜드홍보부]] 신설

### 원칙
- 이식층(명령체계·[[가이드라인 개요|가이드라인]]·프로토콜·본사) = 다음 사업으로 복사 / 도메인층(SOP·학생·CRM) = 에이닷 전용.
- 판단이 필요한 일은 'AI가 대신 결정'이 아니라 '선택지+근거 → 원장 결정'으로 (운영자 성장). `decisions/`·`errors/` = 디리스킹 장부.
