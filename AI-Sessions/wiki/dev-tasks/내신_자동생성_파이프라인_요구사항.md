## 1. 목표 요약

원장의 최대 병목 = **학생별 내신 계획 수립 + 예상문제 세트 제작**.

## 2. 입/출력 정의

### 입력

| 항목 | 소스 | 형식 | 비고 |
|---|---|---|---|
| 학생 기본정보 | 학생_마스터.xlsx | 엑셀 | 이름, 학교, 학년, 목표, 배정교재 |
| 학생 실력 데이터 | 레벨테스트 결과 | 엑셀/수동입력 | 듣기·어휘·구조·독해·언어력 점수 |
| 학교별 기출 | raw/(학교명)/(년도).pdf | PDF (raw 불변) | 3개년 필수, 없으면 프로파일 불가 |
| 시험 범위 | 원장/강사 입력 | 텍스트 | 교과서 단원, 부교재 범위 |
| 학교 출제 프로파일 | sources/학교_출제_프로파일_*.md | MD (ingest 산출물) | 기출 분석 완료 후 존재 |
| 기존 모의고사 DB | 모의고사_문제DB 구글시트 | 시트 | 지문 풀, 유형 태깅 |

### 출력

| 항목 | 형식 | 비고 |
|---|---|---|
| 학생별 내신 계획 (주차별) | PDF/MD | 진도·숙제·예상문제 분량 포함 |
| 학교별 예상문제 세트 | PDF (조판) | 골든레이아웃 적용, 학원 내부용 |
| 출제 프로파일 업데이트 | MD (wiki/sources/) | 시즌마다 갱신 |

## 3. 파이프라인 단계

```
[입력 수집] → [기출 분석 (④)] → [출제 프로파일 생성] → [학생 진단] → [로드맵 생성] → [예상문제 생성 (⑤)] → [조판] → [검수] → [배포]
```

### 단계별 상세

1. **입력 수집**
   - raw/ 에서 기출 PDF 수집 (ingest)
   - 학생 정보는 학생_마스터에서 읽기
   - TODO(원장 확인): 수동 입력 vs 자동 수집 경계

2. **기출 분석 (④)**
   - PDF → 유형 분포, 범위, 변별 영역 태깅
   - 난이도 = 대비가능성 모델 적용 ([[내신_성적향상_로드맵_방법]])
   - 산출: [[템플릿_학교출제프로파일]] 양식 A

3. **출제 프로파일 생성**
   - 학교 × 학년별 프로파일 문서 (wiki/sources/)
   - 암기커버 %, 기본기 %, 변별 % 산출

4. **학생 진단**
   - 레벨테스트 + 수업 성적 + 목표 등급
   - 두 레버: 기본기 수준 × 암기 투자 가능량

5. **로드맵 생성**
   - 전략 매트릭스 적용 → 목표 × 시험 성격 → 경로 결정
   - 산출: [[템플릿_성적향상로드맵]] 양식 B

6. **예상문제 생성 (⑤)**
   - 교과서 본문 변형 + 모아닷 DB 유사 외부지문
   - 유형 분포 맞춰 세트 구성
   - 검수 전 저작권 체크 (학원 내부 수업용 전용)

7. **조판**
   - 골든레이아웃 적용
   - 강의용교안(컬러) + 리뷰테스트(흑백) 동반

8. **검수 ⭐ 자동/수동 경계 = 원장 확인 포인트**
   - TODO(원장 확인): 현재 제안 — ① 분석·생성·조판 = 자동 / ② 로드맵·출제 방향·최종 배포 = 원장 검수

9. **배포**
   - 공유폴더 또는 출력 → [[4_행정운영부]] 핸드오프

## 6. 우선순위 제안

1. **1단계** (즉시): 기출 분석 → 출제 프로파일 자동화 (④ ingest 기반)
2. **2단계**: 예상문제 생성 엔진 (⑤, 교과서 변형 + 모아닷)
3. **3단계**: 로드맵 자동 생성 (전략 매트릭스 자동 적용)
4. **4단계**: 전체 파이프라인 통합 + 검수 대시보드
