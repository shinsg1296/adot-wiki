# 부서 개요 (에이전트 조직도)

학원은 **원장(신상균) + 교무부 부장(선영) + 정식 강사(조정국 / 조하은 2026-07 퇴사 예정) + 조교 4명** 체제입니다(인적 구성 → [[학원 자원]]). 여기서 "부서"는 **에이전트에게 위임하는 역할 단위**이고, 사람(원장·강사·조교)이 그 역할을 AI 에이전트의 보조로 수행합니다.
일을 시킬 때 "○○부 업무로 ~해줘"라고 지정하면 해당 부서 노트의 규칙을 따릅니다.

조직은 **2층 구조**입니다: 사업을 *굴리는* 운영 레이어(1~6) + 사업을 *키우는* [[0_본사_전략실|0_본사(전략실)]]. (근거: [[의사결정_조직구조]])

## 조직도

```

        ↑↓
원장 (대표) ── 운영 레이어 (사업 안에서)
├── 1_학습관리부  ⭐ 수업일지·성적 (매일, 자동화 완성도 높음)
├── 2_교무부        진도·자료·숙제·DB콘텐츠
├── 3_상담소통부    통화·상담·알림

├── 5_시스템개발부  도구·자동화·DB기술 (횡단)
└── 6_브랜드홍보부  HAWE·홍보·신규문의 (입구)
```

각 부서 = Hermes 프로필(독립 에이전트). 원장=임금, 본사=책사. 각 팀장 SOUL.md는 그 부서 위키 규칙을 따른다.

| 2_교무 | **슬기** | `seulgi` | 지혜·분석 / 시험지 분석·성적 로드맵·예상문제 |

| 4_행정운영 | **바름** | `bareum` | 바르고 단정한 운영 / 명단·출결·결제·시간표 |
| 5_시스템개발 | **새롬** | `saerom` | 새로움·창조 / 도구·자동화 엔진 |

## 왜 학습관리부가 1순위인가

- 매일 밤 10시 수업일지 입력 = **가장 빈도 높고 누락 시 학부모 신뢰에 직결**
- 이미 CRM 자동입력 방법이 검증되어 있어 즉시 위임 가능
- 다른 부서 업무(상담, 자료)도 수업일지 데이터를 기반으로 함

## 분담 규칙 (책임 경계)

- **R1 도구 소유**: 엔진(도구) = [[5_시스템개발부]] / 산출물 '내용' = [[2_교무부]]·[[1_학습관리부]]
- **R2 DB 책임**: 콘텐츠 = [[2_교무부]] / 기술(병합·푸시) = [[5_시스템개발부]]
- **R3 외부 채널**: 외부·마케팅 = [[6_브랜드홍보부]]
