# 3. 상담소통부

**미션**: 학부모와의 모든 접점(통화·알림·상담)을 기록하고 관리

## 담당 업무

- 학부모 통화 자동 기록 파이프라인 운영 → [[SOP_통화기록_처리]]
  (Tasker + 녹음 + Gemini 전사 + AppsScript → DB)
- 상담 일정·내용 기록, 면담용 특이사항 축적
- 알림톡 발송 보조 (발송 자체는 원장이 직접)

## 핵심 규칙

- 학부모 대상 문구는 항상 경어, 학생 평가는 구체적 사례 기반
- 면담용 특이사항은 수업일지의 "면담용" 필드와 연동 ([[1_학습관리부]])
- 민감한 상담 내용은 위키가 아닌 CRM에만 기록

## 신규 문의 응대 (전화)

- 신규 문의 전화는 **무조건 원장이** 받는다.
- 확인: **학년 · 학교 · 유입경로** → 간단한 학원 시스템 안내 → **레벨테스트 날짜** 잡기.
- 레테 날짜 잡으면 → **학생 이름** 확인 / 안 잡으면 이름은 안 물음.
- 이후: [[레벨테스트]] 진단·배치([[2_교무부]]) → 등록([[4_행정운영부]]) → OT. (유입 깔때기 ← [[6_브랜드홍보부]])

## 상담 종류 · 트리거

- **입학 상담**.
- **시험 후 선별 연락**(학부모): ① 퇴원 위험군 ② 성적이 너무 안 나온 학생 ③ 성적이 너무 잘 나온 학생.
- **퇴원 위험군 신호**(전부 위험군으로 취급): 성적 급락 · 결석/숙제 미이행 증가 · 태도 변화 · 표정 변화 · 직접 언급.
  - 자동 감지 가능 = 성적·결석·숙제(데이터) / 사람 관찰 필요 = 태도·표정·직접 언급.
- 내용: 주로 **학습 태도 · 다음 계획**. 상담 준비엔 **통화 기록(전사 DB)을 실제로 활용** → 상담 전 브리핑.

## 학부모 소통 (현황 · 개선)

- 기본 채널 = **수업일지** → **CRM 자동 알림톡 발송(됨)**. 단, **전화 소통은 거의 단절**(학생 입을 통해 전달되는 수준) — 개선 과제.
- 방향: ① 수업일지(자동)를 **신뢰 채널로 정착** ② **공지·안내 알림톡 시스템 확장** ③ **월별 정기 설문(학부모·학생)** 꾸준히.
- 설문 = **학부모 만족도(성과 지표) 측정** 수단 → [[기획기준]]. 발송 인프라 → [[학부모_소통_자동화]].
