# 하위큐(HAWE Q) 문제 생성 시스템

## Summary

기출 **지문을 재료로 새 문제(현재 제목추론)를 자동 생성**하고, **웹 picker로 골라 맞춤 문제지·정답해설(docx/pdf)을 뽑는** 시스템. AINGEL식 다운로드 자료의 자체 버전. 교무부 ⑤예상문제 출제의 자동화 엔진이며, 같은 파이프라인으로 다른 연도·9월·수능·**부교재**·다른 **문제 유형**(주제·어법·어휘·요약·순서·삽입)까지 확장하도록 설계. 2026-06-10 기준 **제목추론 144문항(8시험: 23~26학년도 6월 고1/고2 × 18지문) 완성**.

## Context

- **목적**: 평가원 출제원리([[제목추론_출제원리]])를 역산해 기출 지문으로 변형 문제를 대량 생성 → 학원 내부 수업·숙제용 자료. 본사 관점에선 "이식 가능한 콘텐츠 공장"(→ [[이식성_체크리스트]]).
- **데이터 위치**: 코드·중간산출물 전부 `C:\모의고사자동화`. 원천 지문 = Google Sheets DB Q시트(모의고사 ingest 결과, → [[내신_분석_시스템]]).
- **현황**: 제목추론만 데이터 존재(144). picker는 유형 무관 스키마라 다른 유형 추가 시 그대로 수용.
- **향후**: ①유형 6종 추가 ②부교재·타 연도/9월/수능 등록 ③**Netlify 외부 공개**(정적 프론트 + 클라이언트 docx 생성 검토).

## Details

### 파이프라인 (재현 순서)

| 단계 | 스크립트 | 입력 → 출력 | 핵심 |
|---|---|---|---|
| 1 추출 | `_hawe_q_extract.py` | DB Q시트 → `_hawe_q_all.json`·`_hq\q_NNN.json`·`_hq\_index.json` | 순서/삽입 **결정론 조립**, 빈칸 미리채움, 장문 (a~e)마커 제거, 어법/어휘/무관/장문에 **해설(교정 근거)** 첨부, 꼬리표 제거 |
| 2 생성+검증 | `_hawe_q_workflow.js` (Workflow) | `_hq\q_NNN.json` → 블록 | 문항별 파일 read(토큰 절약), 제목문제 출제→검증. **청크 12 + 재시도**(rate limit 회피) |
| 3 병합 | `_hawe_q_merge.py` | 실행 출력(들) → `_hawe_q_merged.json` | 인덱스 기준 커버리지 확인(144/144), `by_idx` 포함 |
| 4 정답 재배치 | `_hawe_q_rebalance.py` | merged → `_hawe_q_rebal.json` | 정답 위치 ①~⑤ 균등(4·4·4·3·3), 선지·해설 라벨 일괄 스왑(내용 불변) |
| 5 빌드 | `_hawe_q_build_all.py` (+ `_hawe_q_build.py`) | rebal → 시험별 8세트 docx/md | `build_munje_docx`(2단·출처 대괄호)·`build_ansol_docx`(정답표+해설), `strip_codes`로 A/B코드 제거, **인덱스-진실 normalize** |
| 6 렌더 | LibreOffice `soffice --convert-to pdf` | docx → pdf | exam_server `_libreoffice_convert`와 동일 방식 |
| 7 뽑기 | `hawe_picker.py` (Flask:5057) + `하위큐_뽑기.bat` | `_hawe_q_rebal.json` → 웹 picker | 시험·유형·검색 필터→선택→문제지+정답해설 ZIP |

### 핵심 교훈 (반드시 지킬 것)

1. **★`지문_복원`은 고쳐진 지문이 아니다** — `지문`의 복사본이라 어법/어휘 정답 위치의 *일부러 틀린 단어*(예 low↔high, confuse↔help, having→have)가 그대로 있다. 그냥 쓰면 오류 박힌 지문. → **해설(=정답으로 고쳐진 한글 뜻)을 근거로 교정**해야 함. (원장 발견)
2. **순서/삽입은 LLM에 맡기지 말고 결정론 조립** — 답 순서 코드(①=ACB…)·삽입 위치로 코드가 미리 합쳐 마커 제거. LLM 순서 오류 방지.
3. **대량 동시 실행 = rate limit** — 288 에이전트 한 번에 → "API Error: Rate limited" 132건. **청크(≈12) + 재시도**로 회피. 실패분만 재실행 후 병합.
4. **에이전트 출력 신뢰 금지, 인덱스(파일명)=진실** — 헤딩을 `### N번` 대신 `**N번**`으로 쓰거나 출처를 틀리게 적는 경우 있음 → `_index.json`으로 N·출처 강제 교정(normalize).
5. **LLM 정답 위치 편향** — 정답이 ②③에 몰리고 ⑤ 거의 없음 → 5단계 후 재배치 필수.
6. 콘솔 한글 인코딩 → 스크립트는 `sys.stdout.reconfigure(encoding='utf-8')`.

### 신규 등록 절차 (다른 시험·부교재 추가 시)

1. 해당 지문을 DB Q시트에 ingest (→ 모의고사 ingest 스킬 / 부교재 ingest).
2. `_hawe_q_extract.py`의 `EXAMS`(또는 소스 목록)에 시험 ID 추가 → 실행.
3. `_hawe_q_workflow.js` 실행(`{dir, count}` 또는 `{dir, indices}`).
4. merge → rebalance → build_all → render.
5. picker는 `_hawe_q_rebal.json` 재로드(서버 재시작)만 하면 신규 문항 노출.
