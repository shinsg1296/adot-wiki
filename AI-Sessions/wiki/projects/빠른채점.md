# 빠른채점 (학생 채점 시스템)

학생이 폰으로 자기 답을 입력→**즉시 채점·기록**되는 시스템. [[하위보카]] 앱 안 iframe으로 열림. 백엔드=**구글 Apps Script**(`grade.html` 템플릿 + `gradeExams`/`gradeScore`/`gradeSubmit`/`gradeMyResults`, 정답키·채점기록=구글시트). [[하위고사_모의고사제작]](모의)·[[하위파이널_내신제작]](내신) 정답키를 학생 채점에 사용. 깊은 기술=메모리 `process_quick_grade`·`process_quick_grade_v2_filter`·`process_student_login_roster`.

## Summary

- **v2 개편(2026-07-06 라이브)**: ①**내신 학교·학년 노출필터** — 학생은 **자기 학교+학년 내신만** 봄(선생님=전부·모의=전원) ②**내신/모의 카테고리** 분리 ③**이름 표기** "26년 1학기 2차 정기고사 [학교][학년]"(내부 키 유지→기존 기록 안 깨짐) ④**문항수 동적**(45 고정 제거→시험별 실제 문항수).
- **신규생 로스터 반영(2026-07-10)**: 새 학생이 로그인+자기 학교 빠른채점 자동 노출되게 갱신. 178명.

## Context — 데이터 · 결정

- 정답키·채점기록 = 구글시트(`GRADE_ID`). 시험목록·문항수는 시트에서 산출. **학생 학교·학년(STU)·시험 분류(EXAM_META)는 gs 코드에 내장** → 시트 안 건드려도 노출/분류 작동(정답 교체와 분리).
- **결정(원장)**: 모의고사 = 전 학생 노출(학년 필터 안 함) / 내신 정답 = 파이널(예상) 아니라 **실제 정기고사 답으로 교체**(원장 제공 예정 — 현재 파이널 기반).

## Details

### 내신 노출 필터
- 학생 id(소문자)→{학교,학년} 맵(STU)·시험별 {cat,school,grade,label}(EXAM_META)를 gs에 내장. 내신은 학생 school+grade 일치할 때만 노출. 미등록(선생님)=전부. **새 학교시험은 EXAM_META에 school/grade 등록 후 gs 재생성**(빠뜨리면 학생에게 안 뜸).

### ★ 신규생 로스터 반영 절차 (재사용)
1. CRM **수강생기본정보 XLS**(학교=col9·학년=col10·아이디·코드) 다운로드.
2. **merge**: 새 명단 + 명단서 빠진 학생은 접속 유지(임의 삭제 안 함) → `_wlogin_full.json`·`_grade_student_map.json`.
3. **양쪽 백엔드 배포 필수**: ①**워드 백엔드 WLOGIN**(로그인, scriptId 19Ls…) ②**grade 백엔드 STU**(빠른채점 필터, scriptId 1NTy…). 로그인=에이닷 **아이디만**(비밀번호 없음).
4. 검증: 로그인 API `?action=login&id=…`→ok+이름 · grade 앱 내신카드 개수.

### 배포법 (clasp 없음 → 브라우저 Monaco)
- script.google.com 프로젝트 열기 → `navigator.clipboard.readText()`(PowerShell Set-Clipboard로 전송) + `monaco…setValue()`로 **로스터/코드 블록만 외과 교체**(djb2 해시로 무손상 검증) → 저장(저장버튼 disabled=완료) → 배포 관리 → 수정 → 버전 **새 버전** → 배포(**URL 유지**). 버전 드롭다운의 "새 버전"은 `find` ref 클릭이 확실(좌표·키보드는 빗나감).
