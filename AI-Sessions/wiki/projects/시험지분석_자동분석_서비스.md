# 시험지분석 자동분석 서비스 (examcheck :8765)

## Summary
학교 시험지 PDF/사진을 업로드하면 **OCR→마스터 원문뱅크 대조→출처·범위·유형·변형 판정→정답·해설→같은유형 파이널·블로그 초안**까지 자동으로 뽑는 셀프서비스 사이트. Flask 서버 `examcheck_server.py`(localhost:8765) + `api_pipeline.py`(Claude opus) + `analyze_exam.py`(매칭엔진, passage_bank.json). 2026-07 세션에 6단계 완성 + 성능/정확도 대폭 개선.

## Context
- 백엔드=구독제 `claude -p`(API 0원·opus)와 로컬 CLI 하이브리드. 터널 접속 시 opus API 강제.
- 인증=패널 비번 `0715`(`_panel_pw.txt`) + HMAC 세션쿠키. 공개는 cloudflared 퀵터널(`_cf_url.txt`).
- 상세 운영본=메모리 project_examcheck_subscription / reference_exam_analysis_build.

## Details

### OCR 속도 최적화 (22분 → 3분, 5배)
- `extract_items` 페이지 **병렬 처리**(ThreadPool 8). 저작권 출력필터 차단 페이지는 세로분할 재귀(`_extract_one`)도 **병렬화** + 깊이3 + 즉시분할(재시도0).
- **필드별 최장값 병합**: 한 조각이 발문만·다른 조각이 지문만 잡아도 합쳐짐(발문·배점 유실 방지).
- `render_png` 스캔사진 긴변 1600px 정규화(과부하 방지).

### 만점 정확도 (84→100)
- 서답형 표기차 중복(`서술형 3`vs`서술3`)=`_canon_num` dedup(만점 110→100).
- **답지에서 공식 배점도 추출**(`extract_answers`가 정답+배점) → 문항 배점을 정답표값으로 교정(만점 항상 정확).

### 출처에 원본 문항번호 부착
- 매칭 결과 출처에 **몇 년도 몇 월 몇 번**·**수특 몇 강 몇 번**까지: `orig_qnum`(raw source의 'N번' 우선 > q필드, 수특 q=Ex02→2번), `with_qnum`. 엔진 로더가 `q`·`src_raw` 보존.
- **포함도(containment) 폴백**: OCR이 잘려 자카드 낮아도(<0.60) 뱅크 원문의 부분집합이면(포함도≥0.90) 그 지문으로 확정(잘린 문장삽입 등 미상 오판 방지). "지문 일부만 OCR됨" 비고 표기.
- 부교재 매칭=`analyze_exam.load_engine`이 passage_bank 전 카테고리 로드 + 동적 scope(하드코딩 고1 제거) + `_canon_src`(수특Light≡수능특강 Light).

### 학생 답안 오버레이 (빠른채점 연동)
- URL `?student=이름&exam=시험` → exam_server `/api/student_answers`(문항별 선택답안) fetch → 그 학교 분석 폼에 **학생이 고른 답 마킹**.
- 오버레이 모드(`window.__STU`, body.stu-mode): 업로드폼·픽커·히어로 숨김·**문항카드 정오색상**·블로그섹션제거·**약점요약**(유형별 + 출처를 모의/부교재/교과서 크게분류→"어디서 틀렸는지"버튼으로 문항번호·원출처 세부).
- 함수=stuVerdict/stuWeakness/qChoices/stuLine/initStuOverlay. alert 금지(창 얼림)→비차단 배너.
- ★번호 일치 확인: 빠른채점 정답키 번호(1~21) = 분석 num(1~21+서술) 정확 대응.

### 진입 링크 (TV 대시보드/빠른채점 → 팝업)
