# 하위고사 — 모의고사 제작 (모의고사 자동화)

## Summary

**모의고사 자동화 = 모의고사 제작 = 하위고사**(원장이 나눈 세션 이름). 모의고사 PDF(문제지·해설지·음원)를 받아 **[[문제DB_공유_데이터베이스|공유 문제 DB]]에 ingest**하고, 그 DB로 모의고사 문제지·답지·빠른채점을 출력하는 계열. 출제 3계열 중 **수능 출제 이해도**가 핵심인 라인.

## Context

- **세션 구분**: 변형=「하위큐」([[하위큐_문제생성_시스템]]), 내신 파이널=「하위파이널」([[하위파이널_내신제작]]), **모의고사=「하위고사」(이 노드/이 세션)**.
- **필요한 이해**: **수능/평가원 출제원리** — [[평가원_21유형_출제원리]]. (내신 특화 아님.)
- **데이터**: 전부 [[문제DB_공유_데이터베이스]]에 누적. 코드=`C:\모의고사자동화`.

## Details

- **ingest**: 새 모의고사 → 작명(시행연도)·파일저장·정답표 파싱·독해/듣기 추출·Q시트 push. 절차·함정 상세 = 모의고사 ingest 스킬(메모리 `skill_mock_exam_ingest` + `C:\모의고사자동화\.claude\skills\mock-exam-ingest\SKILL.md`). 26-06 고1·고2·고3 45/45 완료. **26-07 고3(전국연합) 진행 중(2026-07-15)**: 소스 보관(문제지·해설지·듣기 mp3→모의고사 폴더)·정답45+3점10 확정(3점=문항지 `[3점]` 페이지추적: 6·11·13·24·32·33·34·37·39·41·만점100)·Q시트 표준+인라인+듣기텍스트 **24/45 적재**(드라이버 `_ingest2607*.py`). 잔여=통합시트(빈칸/순서/삽입/요약/장문)·이미지(도표·안내문·듣기그림/도표, Drive)·듣기 16_17·해설/번역·원문뱅크/문제뱅크 등록.
- **출력물**: 모의고사 문제지/답지(exam_server, Word COM) + 듣기음원 병합 + 빠른채점(정답키). 빠른채점 상세는 메모리 `process_quick_grade`. **exam_server(로컬 :5000)는 로그인 시 자동시작·상시가동**(작업 스케줄러 `AdotExamServer5000`, pythonw 런처 `_exam_server_launch.py`). ★코드 수정 후엔 작업 재시작 필수(`Stop/Start-ScheduledTask`). 상세=메모리 `process_local_server_autostart`.
- **★정답 정확도 검증**: 생성/적응 문항(특히 **도표 Q25**)은 DB 등록·배포 전 차트↔선지 수치 대조 검증 필수(2026-06 27특강 8회차 중 도표 5개 오답 발견·수정). 오답=신뢰도 직격. 객관식 해설 생성 시 **멀티에이전트 워크플로(독립 해설생성 + 적대적 정답 재검증)로 기록답을 회의적으로 교차확인** 후 반영(gen·verify·기록 3자 일치만 통과, 의심은 반영 전 별도 보고).
- **★★정답키 조립 매핑 오류(2026-07-15 발견·수정)**: 저장 세트(saved_sets)를 빠른채점 정답키로 조립할 때 **문항 위치와 어긋난 원본 시트 행의 답이 채워질 수 있음** — 지문·선지는 정상(학생이 푸는 문제는 멀쩡)인데 **정답키만 틀려 정답 맞힌 학생이 오답 처리**됨. 27특강 3·4·5회에서 각 3건(총 9건) + 26-07 고3 3점 배점 8건(표준 기본값이 실물 시험지와 불일치) 발견·정정. **★반대로 시트 원본은 거의 깨끗**(27특강·수특독해·부교재·듣기 고위험 643행 전수검수 결과 시트 자체 오류는 1건뿐=`수특독해 10강 3번 Q40` ②→④). → **빠른채점이 틀리면 시트보다 세트 조립 매핑을 먼저 의심.** 전수검수법=문항 내용(지문+선지)↔등록정답 대조(멀티에이전트 gen+적대검증, 둘 다 불일치일 때만 플래그→오탐0)·이미지형(도표·안내문·듣기)은 직접 확인·소배치(75) 순차실행(동시 대량=서버 rate limit). 상세=메모리 `process_grade_key_mapping_audit`. 미검증=1·2회 세트.
