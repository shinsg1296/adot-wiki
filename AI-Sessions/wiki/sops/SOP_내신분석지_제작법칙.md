# SOP: 학교 시험지 AI 분석지 제작 법칙

담당: [[2_교무부]] (엔진·매칭 = [[5_시스템개발부]])

## Summary

스캔한 학교 시험지를 **원문 출처·변형 여부·정답·해설**이 담긴 인터랙티브 웹 분석지로 자동 변환하는 전 과정의 규칙. 문정여고2로 확립(2026-07-01)하고 원장 검수 피드백을 반영해 **7개 시험**(문정여고2·문정여고3·동신고1·동신여고2·동신여고3·고려고1 외)까지 적용하며 다듬었다. 새 학교 시험을 분석할 때 아래 **8개 영역 전부**를 적용한다. 데이터는 위키가 아니라 파일에 있다(코드 `C:\시험지분석\master\`) — 이 문서는 "무엇을 어떤 원칙으로 만드는가"만 둔다.

> 이 문서는 [[내신_분석_시스템]](2026-06 개요, `analysis/`·db_corpus 매칭)의 **후속 아키텍처**다. 현재는 `master\`(원문뱅크 `mock_passages.json` + `_match_{학교}.py` + `_build_web_data.py` → 웹 `data.js`)로 인터랙티브 분석지를 생성한다.

## Context

- **왜 규칙이 필요한가**: 학교는 지문만 그대로 빌리고 선지·밑줄·유형까지 재구성한다. 자동 매칭·정답 유추가 통하지 않으므로, 사람의 판단 기준(공식답지 필수·원문 안 지어냄 등)을 규칙으로 고정해야 분석지가 신뢰성을 갖는다.
- **결정권자**: 원장. 아래 규칙 대부분은 원장 검수 중 지적으로 확립됐다(§근거).
- **저작권**: 학원이 합법 보유한 자료를 **내부 분석·수업 전용**으로만. 재배포·재판매 금지.

## Details

### 파이프라인 5단계
`OCR`(사진→문항별 지문·선지·마커, 공식답지 별도) → `매칭`(원문뱅크 대조로 출처·변형 판정) → `분석 저장`(판정·정답·유사도) → `웹 빌드`(§1~§6 규칙 적용) → `서빙`(로컬, 캐시버스팅).

### 1. 출처·매칭·판정 — "어디서 왔고 얼마나 바뀌었나"
- 출처는 **년·월·번호를 풀어서**: "2024년 6월 고2 모의 20번". 코드형(26-06) 금지. 교과서·부교재는 정식명.
- **구조형(무관·삽입·순서·어법·어휘)은 변형이 아니라 "원본(문항 구성)"**. 마커·재배열·함정은 문제 장치일 뿐. 단어가 많이 바뀌었을 때만 진짜 변형.
- **'어휘 일치'와 '문장 매칭'은 다른 지표**. 재작성 변형은 어휘 50%+여도 문장 대응 0 가능.
- '원문 DB 미보유'는 정말 없을 때만. 뱅크 밖 시중 도서(예: 《넛지》)는 '미상'이 아니라 **책 이름으로** 표기.
- 부교재 단어퀴즈는 '부교재'로, 도표는 '내용일치'가 아니라 **'도표'** 유형으로.

### 2. 데이터 무결성 — 절대 원칙 (틀리느니 비운다)
- **원문은 절대 지어내지 않는다.** 권위 소스(아잉카 한줄해석=실제 모의 원문·시험지 원문)만. 동의어 임의 치환 금지. 확신 없으면 '미상'.
- **정답은 공식 답지가 필수.** 모의 정답·학생 마킹으로 유추 금지(학교가 선지·마커·유형 재구성). 답지 없으면 원장에게 요청.
- **원문 손상은 지문 통째 정본 교체**(부분 수정은 다른 손상 놓침). 탐지=일치 100%인데 특정 문장만 저매칭.
- **지문 본문 화면 통째 출력 금지**(콘텐츠 필터). 화면엔 글자수·유사도·위치만, 인용 15단어 이하.
- **비교는 정규화, 표시는 원문**: 따옴표·대시·하이픈·공백·원문자 마커는 같은 것으로 취급(“high-density”=“high density”).

### 3. 정답·서술형 답안
- 서술형은 **실제 문제·조건·발문 그대로**. 일반 발문·선택형 정답표 오사용 금지. `[N점]`·`[발문]` 중복 라벨 제거.
- **정답·해설·채점기준은 각각 다른 칸.** 정답 칸엔 답만.
- 채점기준표 없으면 원장 위임하에 **매칭 원문+조건으로 모범답안 직접 풀이**(원문 전수 대조 검증 필수), "학교 미제공·원문 기반" 고지.
- 수정공지(정오표)는 별도로 반영(시험지엔 수정 전이 찍힘).

### 4. 시험지 형태 재현 (마커)
- 유형별 실제 형태 복원: 어법/어휘 5밑줄+①~⑤, 무관 문장앞 ①~⑤, 삽입 주어진문장+( ① ), 요약 (A)(B), 빈칸 ____, 순서 (A)(B)(C). ⓐ~ⓔ도 선지번호.
- OCR이 마커 놓치면 **이미지 재확인**. 개수형('틀린 것의 개수')은 선지(1개~5개) 유지.
- 삽입 '주어진 문장'은 **정답 위치에 정답색**("뺀 문장" 아님). 요약문은 별도 박스로 분리. 도표는 텍스트 수치를 **실제 그래프(SVG)**로.

### 5. 표시·레이아웃
- 문항 순서는 실제 시험지대로(선택형→서술형 맨 뒤). 서술형 판정은 유형 아닌 문항번호로.
- 서술형은 뭉치지 말고 구분: 보기=칩 상자, 조건=번호 목록, 하위문항 굵게.
- **지문 공유 세트(16↔서술1 등)는 한 카드로**: [공통 지문 → 각 문항 발문·선지/조건 → 각 정답·해설].

### 6. 원문 대비 (분석의 핵심 가치)
- **항상 문장 정렬 대비**(일치 낮아도 생략 금지): 문장 넘버링+유사도+단어 하이라이트(빨강=바뀜·점선=추가·취소선=삭제).

### 7. QA·검수·재발방지
- 에이전트가 형식·정합 오류를 스스로 찾아 고치는 루프. **진짜 오류는 파이프라인 수정(재발방지) 우선**, 데이터 수정 차선. 인쇄 오타 등 원본 그대로인 건 유지.
- **★OCR 맥락 검증(원문 대비)**: 스캔·학생 필기 겹침으로 OCR이 단어를 오독함(예: her→hey). 매칭도 높은 문항에서 **원문과 다른 단어**를 추출(`_verify_ocr_context.py`, 따옴표·대시·마커·발췌 노이즈 필터) → LLM이 **의도적 변형(어휘 동의/반의 교체·어법 함정·재배열) vs OCR 오류(넌센스 오독)** 판별. ★핵심: **OCR은 없는 접미사·굴절·접두사를 만들지 않음** → `forwardly`·`are`·`irrelevant`는 오독이 아니라 학교가 심은 함정(그대로). 진짜 오독만 **원문 단어로 교정**, 애매하면 **해당 영역 재확대 크롭 재OCR**. 새 학교 분석마다 실행.
- 하위큐 병합은 **(학교+학년+회차) 정확 타깃**(전체 학교시험 제거식 금지).

### 8. 하위큐·문제뱅크·배포
- 분석한 시험을 학생 풀이용(하위큐)·통합 문제뱅크로 전파. 카테고리=학교→학년→회차 캐스케이드.
- **배포는 허락 먼저**(허브·Netlify·공유시트). 현재 크레딧 사정으로 배포 동결, 로컬 빌드만. → [[배포_승인_규칙]]
