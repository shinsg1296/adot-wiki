# SOP · 블로그 시험분석 글 제작·발행

## Summary

내신 시험 분석 결과를 **네이버 블로그 시험분석 글**로 만드는 표준 절차. 흐름은 **분석 완성 → 카드뉴스(이미지) → 초안(.md) → 발행기 자동입력 → 원장 검토·발행**. 핵심 원칙 3가지: ① **저작권 = 집계 수치만**(지문·정답·조건 원문 절대 금지) ② **이모지 없이 사람 손글 느낌**(AI 티 최소화) ③ **발행 최종 승인 = 원장**(자동 발행 금지). 관련 구축 계획은 [[블로그_자동화]], 담당 [[6_브랜드홍보부]].

## Context

- 신규 유입 2위 = 블로그, 네이버 플레이스 상위노출 유지에도 주기 업로드가 핵심. 학교별 내신 분석 = 롱테일 승부처(그 학교 학부모만 검색 → 경쟁 약·전환 최고).
- 벤치마크(타지점 인기글) 공식: 인사+시리즈 예고 → 분석지 → **수치 포인트(총문항·객/서·출처비율·배점·1등급 조건)** → "저작권상 문항 공개 불가" 명시 → 유형별 근거 → 직전 모의 소개 → 전문성 마무리.
- 2026-07-04 국제고1 기말로 **카드뉴스 8장 + 초안 + 발행기 자동입력(이미지 9장 포함)**까지 첫 완성. 이 문서가 그 재현 절차.

## Details

### 1) 파이프라인 (5단계)

1. **시험 분석 완성** — 시험지 분석 엔진(`C:\시험지분석\master`)이 문항별 출처·유형·배점·정답 산출 → `web/data.js`(__EXAMS). 근거는 [[SOP_내신분석지_제작법칙]].
2. **카드뉴스 제작** — 자기완결 HTML → **PNG 이미지가 최종 산출물**(HTML은 수치·문구 수정 시 재생성용 원본). `C:\블로그자동화\카드뉴스\`.
3. **초안(.md) 작성** — `C:\블로그자동화\drafts\NN_학교_학년_시험_분석.md`. 네이버 문체 + `[이미지]` 마커 + `> place:`.
4. **발행기 자동입력** — Electron 발행기(`C:\DAF`)가 초안을 읽어 네이버 스마트에디터에 제목·본문·구분선·**이미지**·장소를 자동 타이핑/붙여넣기.
5. **원장 검토·발행** — `[원장 입력]` 채우고 검토 후 **원장이 직접 발행 클릭**.

### 2) 카드뉴스 표준 구성 (8장, 국제고1 기준)

커버 → **시험범위** → 문항구성·배점 → **배점 유형 TOP 3** → **서술형 출제 원리** → 교재별 비중 → **1등급 컷** → CTA(연락처). 다른 학교도 이 골격 재사용, 데이터만 교체.

### 3) 데이터·내용 원칙 (★저작권 생명선)

- **집계 수치만**: 문항수·배점·유형비중·교재비중·지문수·1등급컷. **지문 원문·정답·서술 조건 원문은 0.** 위반 시 저작권.
- **시험범위 = 교과서·부교재·모의 동일 비중**으로 나열(특정 교재만 크게 강조 금지, 원장 지시). 각 **지문수 병기**.
- **모의고사 지문수 세는 법**: 독해범위(고1=20~42번) 중 학원이 뺀 번호 제외(예 25~28 도표·일치·안내문), **장문 세트(41·42)는 1지문** → 회당 18지문 × 회차. (예: 23·24·26년 6월 3회 = 54지문. 2025 제외 시 명시.)
- **서술형 = 문법 카테고리만**(to부정사·가목적어/진목적어·as 원급 비교 등). 조건 문구·모범답안 영어는 금지.
- **1등급 컷** = 직전 시험 실측 + "비슷할 가능성" 예상. 실제 비율/컷은 `[원장 입력]`.
- 과장·"100%·완벽·성적보장" 배타표현 금지. → [[금지표현]] · [[톤앤매너]].

### 4) 서식 규칙 (카드뉴스·썸네일)

- **이모지 절대 금지** — 📌📊 등은 AI 제작 티의 핵심 신호(원장). 섹션 마커는 짧은 룰선+볼드로.
- **한글 줄바꿈** = `word-break:keep-all` + `text-wrap:pretty`(단어 중간 "분석했습니/다." 방지).
- **로고** = 실제 에이닷 PNG 임베드(다시 그리지 말 것) + flex 카드에선 `align-self:flex-start`+`width:auto`(가로 늘어남 방지). 컬러 = 네이비 `#011337` + 오렌지 `#F07D00`(테라코타는 HAWE 전용, 에이닷 아님).
- **썸네일 대형 텍스트** = 컨테이너쿼리 `cqw` + `white-space:nowrap`(폭 좁아도 글자 안 쪼개짐). 필수 문구 = "YYYY년 1학기 N차 정기고사 / 학교학년 / N차 정기고사 분석".

### 5) 초안(.md) 규칙

- **문체** = 짧은 문장 + 빈 줄(네이버 모바일), 이모지 없는 소제목, 키워드 **"일곡동 영어학원" 5회+** · "에이닷 영어학원 광주일곡지점" 반복.
- **이미지 마커** = 줄 단독으로 `[이미지] 썸네일` / `[이미지] 카드1` … `[이미지] 카드8`(발행기가 해당 PNG를 그 자리에 붙여넣음).
- **`> place:` 줄엔 상호명만** — 메모·주석 붙이면 그게 장소 검색어로 들어가 "검색결과 없음"으로 멈춤. 일곡점 검색어 = `에이닷 영어학원 일곡점`.
- `[원장 입력: …]` 마커로 원장만 아는 값(1등급 비율·우수자) 표시.

### 6) 발행기 운용 (기술 메모)

- 재기동 = `DAF_DEV=1` + `--disable-gpu --disable-gpu-compositing --disable-software-rasterizer`로 Electron 기동(GPU 크래시 회피, `persist:naver` 로그인 유지).
- **★재입력 전 반드시 재기동** — clearField(Ctrl+A+Delete)가 **이미지 컴포넌트는 못 지워** 재입력 시 이미지가 누적됨(9→16). 재기동하면 "작성 중인 글" 취소=폐기로 백지에서 다시 채움.
- **이미지 작게·가운데** = 본문 폭보다 작은 이미지는 네이버가 원본 크기로 가운데 정렬 → 카드 PNG를 **500px 폭**으로 리사이즈(고해상 1080 원본은 `images\_full1080\` 백업, 인스타 재사용).
- 검증 = 개발 엔드포인트 `/api/_devinspect`의 `images` 카운트(9), `placePanelOpen`(false=장소 성공).
