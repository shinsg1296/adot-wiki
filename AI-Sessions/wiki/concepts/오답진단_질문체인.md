# 오답 진단 질문 체인 (재학습 지점 찾기)

목적: AI테스트·오답을 **단순 채점에서 끝내지 않고**, "어디서 이해가 끊겼는지"를 질문 체인으로 분해해 그 지점부터 재학습시킨다. ([[2_교무부]] 검사·분석 IP, 다음 사업 이식 가능)

## 원리

한 문항을 맞추는 데 필요한 **선행 지식을 단계로 쪼개고**, 각 단계를 "아는가?"로 물어 막힌 곳을 찾는다. **대답 못 하는 첫 질문 = 재학습 시작점.**

## 예시 — 전명구 위치 문제

- 문제: "전명구가 동사를 수식할 경우 전명구의 위치는?" / 보기: 문장 맨뒤 · 맨앞 · 동사 바로 앞 · 조동사 뒤
- 학생 오답 흐름: 맨앞(X) → 동사 앞(X) → 맨뒤(O)
- **진단 질문 체인**:
  1. 전명구가 무엇인지 아는가?
  2. 전명구가 수식하는 것이 무엇인지 아는가?
  3. 전명구가 동사를 수식할 때의 품사를 아는가?
  4. 그때의 위치가 어딘지 외웠는가?
- → 막히는 첫 질문부터 다시 공부.

## 검사·조정과의 연결

- AI테스트 80% 미만 / 재시험 = **이해 부족 신호** → 단순 재시험 전에 이 체인으로 원인 진단.
- 3원인(집중도 / 내용 난이도 / 문제 이해력)과 결합 → [[교무부_운영룰]] #2 조정 기준표·#3 검사.

## 개발 연계 (에이전트 간 요청)

- 현 AI 문제풀이는 이 솔루션을 주지 않음 → [[AI진단_문제풀이_솔루션]] 필요.
- 흐름: **교무부 에이전트가 진단 로직(질문 체인) 설계 → 개발부 에이전트가 구현.** (교무부가 "이런 과정 필요" 요청을 개발부에 전달 — R1 핸드오프의 실제 사례)
