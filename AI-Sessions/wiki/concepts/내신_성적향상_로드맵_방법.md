# 내신 성적향상 로드맵 방법 (교무부 ④·⑤ · 핵심 IP)

[[2_교무부]]의 핵심 기능 ④(분석·전략)·⑤(예상문제 출제). **학교별 시험지를 분석해 ⓐ 학생별 성적향상 로드맵을 세우고 ⓑ 패턴 유사 예상문제를 출제하는 방법.** 방법(양식)은 다음 사업으로 이식되고, 내용(학교·학생 데이터)만 에이닷에 남는다 → [[0_본사_전략실]] 이식층 IP.

## 엔진 (4스텝 루프)

1. **분석** — 학교별 기출 → 출제 프로파일 구축 ([[템플릿_학교출제프로파일]])
2. **로드맵** — (프로파일 요구능력) − (학생 현재능력) = 갭 → 주차별 플랜 ([[템플릿_성적향상로드맵]])
3. **실행** — 진도·숙제·**예상문제(⑤)** 로 갭 메우기 (교무부 #1·#2·#6 + 예상문제 출제)
4. **보정** — 리뷰테스트·모의 성적으로 측정 → 로드맵 수정 → 다시 1

## 난이도 = '대비 가능성' (오답률 대체)

작은 학원은 오답률(모집단 통계)을 가질 수 없다. 그래서 난이도를 **점수 획득 경로**로 본다:

| 영역 | 무엇 | 누가 잡나 | 천장 |
|---|---|---|---|
| 암기 커버 | 교과서 본문·범위 내 변형·반복 패턴(필기 있으면 정밀화) | 무작정 암기로도 | 1등급 하위 ~ 2등급 상위 |
| 기본기 필수 | 외부지문·응용·어휘/문법/독해 기본 | 기본기 있어야 | 100점 / 1등급 안정 |
| 변별 | 함정·과추론·운 | 둘로도 어려움 | 최상위 |

→ **난이도 = (기본기 필수 + 변별) 비중** = "암기 천장이 어디서 막히나".

## 전략 매트릭스 (로드맵의 척추)

목표 등급 × 시험 성격 → 경로:

| 목표 ＼ 시험 | 암기형(암기커버 高) | 기본기형(외부지문 高) |
|---|---|---|
| 2등급 | 암기 집중으로 효율 달성 | 기본기 일부 필수, 천장 주의 |
| 1등급/100 | 암기 + 기본기로 변별 돌파 | 기본기 올인(암기 무의미) |

→ 정직한 ROI 제시(과장 금지 → [[금지표현]]).

## 학생 진단 (두 레버)

기본기 수준(어휘·문법·독해) × 암기 투자 가능량(시간·성실). `이 위치 + 시험 프로파일 + 목표 = 전략 확정`.

## raw 입력 (불변)

| raw | 필수? | 용도 |
|---|---|---|
| 학교별 기출 시험지 PDF(직접 스캔, **3개년**) | **필수** | 유형 분포·범위·변별 식별 + ⑤ 패턴 추출 |
| 학교별 수업 필기(학생 사본) | 옵션 | 암기커버 정밀화(없어도 동작) |

→ `AI-Sessions/raw/`에 보관(수정 금지) → `ingest` → 양식 A. (자주 못 받는 필기는 받을 때만 보강.)

## ⑤ 예상문제 출제 엔진 (족집게 대비)

3개년 기출 분석으로 패턴을 잡아 **유사 예상문제를 대량 생성·드릴**한다. (교무부 ⑤)

1. **패턴 추출 (3개년)** — 반복 출제 포인트·유형 분포·지문 출처·서술형 패턴 → 출제 프로파일 강화
2. **예상문제 생성** — 교과서 본문 변형(어법·빈칸·순서·요약·서술형) + 문항DB(모아닷)에서 패턴 유사 외부지문 선별 + 유형 분포대로 세트 구성
3. **드릴** — 학생 반복 풀이 → 빠른채점/exam_server 채점 → 결과가 4.보정 연료

엔진은 기존 모의자동화·답지생성·exam_server·골든레이아웃 응용 (R1: 교무부=출제 설계, [[5_시스템개발부]]=생성 엔진).

**안전장치**
- 생성 문제는 정답 유일성·오류·난이도 **검수 후 배포** ([[리뷰기준]], AI 생성 → 원장 확인).
- '족집게'는 *내부 역량*이지 외부 적중 보장 아님 → 마케팅 표현 주의 ([[금지표현]] · [[6_브랜드홍보부]]).
- **저작권**: 교과서 본문·기출 재가공은 **학원 내부 수업용 전용** (기본 정책, 원장 변경 가능).

→ 루프 완성: **④ 분석 → ⑤ 예상문제 → ③ 검사·드릴 → 성적 측정 → 보정(④로 환류)**. 족집게 정확도가 시즌마다 복리로 향상. (원문자 ①~⑤=[[2_교무부]] 기능, 보정은 ④로 되돌아가는 엔진 스텝)

## 자동화 훅 (R1)

기출 PDF → 출제 유형 자동 태깅 + 예상문제 생성은 기존 모의고사 자동화·답지생성 파이프라인 응용 ([[5_시스템개발부]]). 분석·생성은 자동화, **로드맵·출제 판단은 원장**(선택지+근거 → 결정, 운영자 성장).

## 미래: 자체 오답률 proxy

학원 학생들의 리뷰테스트·내신 결과가 쌓이면 "우리 애들이 어디서 틀리나" = 작은 학원만의 모집단 신호 → IP.
