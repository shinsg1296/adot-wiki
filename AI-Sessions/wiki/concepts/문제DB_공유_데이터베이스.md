# 문제 DB (공유 데이터베이스) — 모아닷 / Q시트

## Summary

학원 영어 문항의 **단일 원천 데이터베이스**. Google Sheets 문항 DB(탭 `Q01`~`Q45` + 통합/이미지 시트), 모의고사·부교재 **ingest 결과**가 누적되는 곳. 코드·중간산출물은 `C:\모의고사자동화`. **출제 3계열([[하위고사_모의고사제작]]·[[하위파이널_내신제작]]·[[하위큐_문제생성_시스템]])이 전부 이 한 DB를 원료로 쓴다** — 이 노드는 그 공유를 명시적으로 묶는 허브다.

> ★용어 정리: 같은 대상을 "DB Q시트 / 모아닷 / 모의고사 DB"로 불러왔다. 전부 **이 노드**를 가리킨다. (내신 시험지 분석 코퍼스 [[내신_분석_시스템]]=`C:\시험지분석`은 **다른 것** — 학교 기출 분석·출처 매칭용. 혼동 금지.)

## Context

- **무엇**: 발문·지문·선지·정답·해설·단어·이미지URL·지문_복원 등 문항 메타가 든 시트. 신규 시험은 `{ID} Q{N}` 행으로 append.
- **어떻게 채우나**: 모의고사 ingest 스킬(메모리) + 부교재 ingest(수특독해·수특듣기 등). → 콘텐츠=[[2_교무부]] / 병합·푸시 기술=[[5_시스템개발부]] (R2).
- **누가 쓰나(공유의 핵심)**: 아래 3계열 전부. 한 곳을 고치면 세 계열에 영향 → DB 정합성(특히 정답 정확도)이 곧 세 산출물 신뢰도.

## Details

### 3계열 ↔ DB 매핑

| 계열(세션) | 노드 | DB 사용 방식 | 필요한 출제 이해 |
|---|---|---|---|
| 모의고사 제작 (하위고사) | [[하위고사_모의고사제작]] | 시트에 **채워 넣고**(ingest) 그대로 모의고사·답지·빠른채점으로 출력 | **수능** 출제원리 |
| 파이널고사 제작 (하위파이널) | [[하위파이널_내신제작]] | 시험범위 지문을 DB에서 **꺼내** 내신 파이널로 역설계 | **내신** 출제원리 |
| 변형문제 제작 (하위큐) | [[하위큐_문제생성_시스템]] | DB 지문을 **재료로 변형문제 대량 생성** | **수능+내신 둘 다** |

### 운영 규칙
- **출제 전 지문이 필요하면 새로 요청하기 전에 이 DB부터 확인**(이미 ingest된 지문 재사용). → 메모리 규칙과 동일.
- **정답 정확도**: 생성 문항(특히 도표)은 DB 등록 전 검증. 오답이 들어가면 3계열 모두 오염.
